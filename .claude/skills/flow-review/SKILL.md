# 유저 플로우 검수

전달받은 유저 플로우에서 **빠진 것**을 찾는다. 코드 리뷰가 아니라 *제품 플로우* 리뷰.

## 1) CRUD 완전성 (엔티티별)
각 핵심 엔티티에 빠진 동작이 없는지:
- **Create** — 생성 진입점·입력 검증·성공/실패 피드백
- **Read** — 목록 + 상세, 빈 목록 상태, 페이지네이션/필터/정렬
- **Update** — 수정 진입·낙관적/실패 처리·권한
- **Delete** — 삭제 확인·되돌리기/소프트삭제·연관 데이터 처리

> 생성만 있고 수정/삭제/목록이 없는지, 상세는 있는데 목록 진입이 없는지 등을 **표로** 점검.

## 2) 사용자 관점 누락 플로우
- **상태 3종**: 로딩 / 에러 / 빈 상태가 각 화면에 있는가
- **인증·권한**: 비회원→로그인 유도, 권한 없는 접근, 세션 만료
- **진입·이탈**: 딥링크/뒤로가기/새로고침, 미완성 입력 이탈(자동저장·경고)
- **에러 회복**: 실패 후 재시도 경로, 네트워크 오류
- **엣지**: 0개/1개/대량, 긴 텍스트, 동시 편집, 중복 제출
- **온보딩/첫 사용**: 데이터 없는 신규 유저의 첫 화면
- **확인·되돌리기**: 파괴적 행동의 확인, undo

## 3) 일관성
- 같은 행동이 화면마다 다르게 동작하지 않는가
- 막다른 길(다음 행동 없는 화면)이 없는가

## 출력
- **CRUD 매트릭스** (엔티티 × CRUD, 누락 표시)
- **누락 플로우 목록** — 영향도 순(🔴 핵심 누락 / 🟡 권장 / 🟢 개선)
- 각 항목에 "왜 사용자에게 문제인지" 한 줄
- 도메인 정책이 필요한 판단은 추측 말고 질문(`domain.md` TODO 인지).
